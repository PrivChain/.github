# Contributing to PrivChain

Thanks for contributing.

## Contribution Areas

- Protocol and integration code
- SDK and MCP tooling
- Tests, reliability, and security hardening
- Documentation and developer experience

## Workflow

1. Open an issue first for non-trivial changes.
2. Fork and create a focused branch.
3. Keep changes scoped and well-tested.
4. Open a PR with clear context and risk notes.

## Pull Request Expectations

- Clear description of what changed and why
- Test evidence (automated and/or manual)
- Security implications noted when relevant
- Backward-compatibility impact stated

## Security and Disclosure

Do not post vulnerabilities publicly. Follow `SECURITY.md`.

## Pre-release Context

Some launch-critical internals are intentionally non-public at this stage. Contributions should align with current disclosure boundaries and maintainers' guidance.
